# MI PRIMER RESPOSITOTIO DE GIT Y GITHUB

##  ¿Qué aprendí del CURS?
### QUE ES GIT?
Git es un sistema de control de verisones de software que permite registrar todo el historial de ca,bios de un proyecto
##### Que es un REPOSITORIO?
Un repositorio es un proyecto que es seguido por GIT y ya tiene un historial de GIT en el que se estan registrandi sus datos.
##### Que es un COMMIT?
Un commit es todo cambio que se le hace al repositorio y estos commits se van guardando en el historial de GIT.
##### Que son las ramas? 
Son ramificaciones del proyecto se podria decir que son como nuevos caminos que va tomando el Repositorio.
##### LA RAMA MASTER
La rama master es donde esta el Repositorio que sale para el Publico.
##### CLON
Es la copia exacta del repositorio cada uno de los desarroladores tiene una copia en su computadora local
#### Fork
Proyecto completamente diferente que se crea apartir de otro proyecto

##  Comandos que usé

- mkdir: Sirve para crear un a carpeta vacia git init:  sirve para crear un nuevo Repositorio.

- echo: Sirve para añadir archivos rapidamente sin tener que abrir un editor.

- git commit -m: Sirve para guardar un cambio o commit que se le hixo al repositorio.

- git push: Sirve oara subir los ca,bios (commits) al repositorio en Github

- git pull: Sirve para decargar los cambios nuevos del repositorio
